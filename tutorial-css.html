1

Variables CSS — el "catálogo de valores"

Una variable CSS guarda un valor (color, número, lo que sea) bajo un nombre que empieza con --. Después la usas con var(--nombre). Si cambias la variable, cambia todo lo que la usa. Esto es exactamente lo que hace tu portal con --cmp-blue.

Mueve el color. Las 6 figuras de la derecha usan TODAS la misma variable como background. Cambias UN valor, cambian TODAS.

:root { --demo-color: #0066CC; } .swatch { background: var(--demo-color); }
En tu portal: tienes --cmp-blue, --cmp-gold, --cmp-dark definidas al principio del CSS. Cada vez que ves var(--cmp-blue) en una regla, el navegador lo reemplaza por #0066CC. Limpio y centralizado.
2

Padding — el espacio interior

El padding es el "colchón" entre el borde del elemento y su contenido. Más padding = el elemento respira más. Sin padding, el texto queda pegado a los bordes y se ve apretado.

La zona naranja punteada es el padding. El azul es el contenido real.

CONTENIDO
.boton { padding: 20px 30px; /* vertical | horizontal */ }
Atajos: padding: 20px = 20px en los 4 lados. padding: 20px 30px = vertical 20, horizontal 30. padding: 10px 20px 30px 40px = arriba, derecha, abajo, izquierda (sentido del reloj desde arriba).
3

Border-radius — redondear esquinas

Suaviza las esquinas del elemento. 0px = esquinas en 90 grados (cuadrado). Un número alto = píldora. 50% = círculo perfecto (si el elemento es cuadrado).

CMP
.tarjeta { border-radius: 8px; }
En tu portal: tu botón .cmp-nav-cta usa border-radius: 8px. Tus tarjetas de regiones usan radios más grandes (12-16px) para sentirse más amigables. Más radio = más "suave"; menos radio = más "serio/corporativo".
4

Box-shadow — sombras que dan profundidad

Hace que el elemento "flote" sobre el fondo. Tiene 4 ingredientes: cuánto se desplaza la sombra en X, cuánto en Y, qué tan difuminada está, y qué tan oscura es.

SOMBRA
.tarjeta { box-shadow: 0 6px 20px rgba(0,0,0,0.25); }
Tip de diseño: sombras sutiles (opacidad menor a 0.15) se ven profesionales. Sombras muy oscuras se ven amateur. Tu navbar usa opacidad 0.08 — casi nada, pero suficiente para separarlo visualmente del contenido.
5

Flexbox — alinear elementos en fila

Cuando un contenedor tiene display: flex, sus hijos se ponen en fila automáticamente. Después decides cómo distribuirlos con justify-content (eje horizontal) y align-items (eje vertical).

A
B
C
.contenedor { display: flex; justify-content: flex-start; align-items: center; }
Regla mental: justify-content = eje principal (horizontal por defecto). align-items = eje cruzado (vertical por defecto). Tu .cmp-navbar usa space-between para mandar el logo a la izquierda y los links a la derecha.
6

:hover y transition — animar al pasar el mouse

:hover es un estado: "cuando el cursor está encima". transition hace que los cambios entre estados ocurran suavemente, no instantáneamente. Ajusta la duración y pasa el mouse por el botón.

Pasa el mouse sobre el botón "Postular ahora" →

0s = cambio brusco e instantáneo (se ve amateur).
0.2-0.3s = profesional, casi imperceptible pero suave.
más de 1s = lento y molesto, el usuario lo siente como lag.

.boton { background: #0066CC; transition: all 0.25s ease; } .boton:hover { background: #002F6C; /* azul más oscuro */ transform: translateY(-2px); /* sube 2px */ }
En tu portal: exactamente este patrón está en .cmp-nav-cta:hover. El botón cambia de color al pasar el mouse, y la animación se ve suave gracias al transition: all 0.2s.
7

Media queries — diseño que se adapta

Un @media aplica reglas solo bajo cierta condición — típicamente el ancho de la pantalla. Así un mismo sitio se ve bien en computador, tablet y celular sin escribir tres versiones. Mueve el slider para "achicar" la pantalla simulada.

Más de 700px → 3 columnas.
Entre 500 y 700px → 2 columnas.
Menos de 500px → 1 columna apilada.

TARJETA 1
TARJETA 2
TARJETA 3
/* Por defecto: 3 columnas */ .grid { grid-template-columns: repeat(3, 1fr); } /* Pantallas menores a 700px: 2 columnas */ @media (max-width: 700px) { .grid { grid-template-columns: repeat(2, 1fr); } } /* Menores a 500px: 1 columna */ @media (max-width: 500px) { .grid { grid-template-columns: 1fr; } }
En tu portal: al final del CSS (líneas 1584+) tienes varios bloques @media (max-width: 600px) que reorganizan el navbar, el hero y el mapa para que se vean bien en celulares. Es exactamente el mismo mecanismo que estás viendo aquí, solo que con tu propio contenido.
